# SEO Search Console Runbook

This runbook is the operational checklist for Google-side SEO setup.

## 1) Verify property

1. Open Google Search Console.
2. Add a **Domain property** for your production domain.
3. Verify via DNS TXT record.
4. Wait until the property is marked as verified.

## 2) Submit sitemaps

Submit:

- `https://YOUR_DOMAIN/sitemap/0.xml`
- Any additional chunk files generated by the app (`/sitemap/1.xml`, `/sitemap/2.xml`, ...).

You can quickly verify sitemap reachability locally/production:

```bash
npm run seo:verify -w nextjs-firebase
```

Set a custom base URL:

```bash
SEO_BASE_URL="https://YOUR_DOMAIN" npm run seo:verify -w nextjs-firebase
```

## 3) Create initial indexing queue

Generate priority URLs from sitemap:

```bash
SEO_BASE_URL="https://YOUR_DOMAIN" npm run seo:priority-urls -w nextjs-firebase
```

Use the output list in Search Console URL Inspection for first-wave requests.

Recommended first-wave order:

1. Home (`/en`, `/fa`)
2. Top city hub pages
3. Top category landing pages
4. High-quality ad detail pages

## 4) Weekly checks

- Coverage: monitor exclusions and errors.
- Search results: review queries and pages by country and locale.
- Page experience/Core Web Vitals: prioritize fixing any regressions.
- Sitemaps: confirm last read date updates and submitted count remains healthy.

## 5) Monthly actions

- Re-submit any new sitemap chunks if volume grew.
- Refresh indexing queue for newly added category/city pages.
- Update title/description templates for pages with high impressions and low CTR.
